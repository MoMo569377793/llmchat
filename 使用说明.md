# llmchat 使用说明

这是一个独立的小工具，只做两件事：

- 用 `chat_with_config.py` 按配置文件直接和模型对话
- 给你一个最小 SDK 示例，方便你不看项目代码也能直接调用

## 1. 目录

```text
llmchat/
├── chat_with_config.py
├── minimal_openai_sdk_example.py
├── config/
│   ├── config.example.json
│   └── config-ikun.example.json
└── 使用说明.md
```

## 2. 为什么 `chat_with_config.py` 会比官方示例长

因为官方示例只展示了最简单的一条调用，而这个脚本还额外处理了这些现实问题：

- 从 `config` 读取参数
- 从环境变量读取 API Key
- 支持交互式多轮对话
- 支持 `chat/completions` 和 `responses`
- 支持不同后端的思考强度字段差异
- 支持重试
- 支持 `--verbose` 查看真实 payload

如果你只想看最简单的写法，就直接看：

- `minimal_openai_sdk_example.py`

## 3. 先准备什么

你只需要：

1. 有 `python3`
2. 如果要跑最小 SDK 示例，安装 `openai`
3. 在 shell 里设置环境变量

例如：

```bash
export IKUNCODE_API_KEY='你的 ikun key'
```

## 4. 最短上手

### 4.1 交互式对话

```bash
python3 chat_with_config.py --config config/config-ikun.example.json
```

### 4.2 单次提问

```bash
python3 chat_with_config.py \
  --config config/config-ikun.example.json \
  --message "你好，请介绍一下自己。"
```

### 4.3 最小 SDK 示例

```bash
python3 minimal_openai_sdk_example.py
```

## 5. config 最少需要哪些字段

最小可用配置：

```json
{
  "llm": {
    "enabled": true,
    "label": "your-model",
    "provider": "openai_compatible",
    "base_url": "https://api.example.com/v1",
    "api_key_env": "YOUR_API_KEY",
    "model": "your-model-name"
  }
}
```

字段说明：

- `enabled`
  - 是否启用
- `label`
  - 显示名称
- `provider`
  - `openai_compatible` 或 `openai_responses`
- `base_url`
  - API 根地址
- `api_key_env`
  - 环境变量名，不是明文 key
- `model`
  - 模型名

## 6. 哪些是常用可选项

- `temperature`
- `timeout_seconds`
- `max_output_tokens`
- `model_reasoning_effort`
- `model_output_effort`
- `model_thinking_level`

说明：

- GPT 类常用 `model_reasoning_effort`
- 如果某些后端用别的名字，也可以单独写

## 7. 现成的 Ikun 示例

文件：

- `config/config-ikun.example.json`

它已经是一个可直接改的模板。

## 8. 如何接入其他模型

最简单的方法：

1. 复制模板

```bash
cp config/config.example.json config/config-my-model.json
```

2. 至少改这几个字段

- `label`
- `provider`
- `base_url`
- `api_key_env`
- `model`

3. 如果这个模型支持思考强度，再补一个相关字段

常见情况：

- GPT 类
  - `model_reasoning_effort`
- Poe Claude
  - `model_output_effort`
- Poe Gemini
  - `model_thinking_level`

## 9. `--verbose` 有什么用

```bash
python3 chat_with_config.py \
  --config config/config-ikun.example.json \
  --message "你好" \
  --verbose
```

它会打印：

- 实际请求的 URL
- 实际发送的 payload 预览

适合用来确认：

- 到底走的是哪个 endpoint
- 你的思考强度字段到底有没有发出去

## 10. 多行 message 怎么输入

### 方法 1：直接换行

```bash
python3 chat_with_config.py \
  --config config/config-ikun.example.json \
  --message "请帮我完成以下任务：
1. 总结这段话
2. 提取重点
3. 给出建议"
```

### 方法 2：Bash 的 `$'...'`

```bash
python3 chat_with_config.py \
  --config config/config-ikun.example.json \
  --message $'请帮我完成以下任务：\n1. 总结这段话\n2. 提取重点\n3. 给出建议'
```

### 方法 3：从文件读入

```bash
python3 chat_with_config.py \
  --config config/config-ikun.example.json \
  --message "$(cat prompt.txt)"
```

## 11. 交互模式命令

- `/exit` 或 `/quit`
  - 退出
- `/clear`
  - 清空历史

## 12. 常见问题

### 12.1 为什么 `api_key_env` 不是直接写 key

因为更安全，也更方便换 key。

正确：

```json
"api_key_env": "IKUNCODE_API_KEY"
```

### 12.2 为什么默认不限制输出长度

因为这个小工具的目标是“尽量直接地验证模型”。

如果你要限制，再手动加：

```bash
--max-output-tokens 1200
```
