# llmchat 使用说明

这份说明按一个最实用的顺序来写：

1. 刚下载下来先做什么，怎样 1 分钟内跑出结果
2. 最小实例的代码到底由哪些部分组成
3. 如何上手更完整的 `chat_with_config.py`
4. 复杂实例比最小实例多了什么，为什么需要这些内容

## 1. 刚下载下来先做什么

你现在目录里有这些文件：

```text
llmchat/
├── chat_with_config.py
├── minimal_openai_sdk_example.py
├── config/
│   ├── config.example.json
│   └── config-ikun.example.json
└── 使用说明.md
```

如果你是第一次用，建议先跑**最小实例**，因为它最容易立刻出结果。

### 1.1 安装依赖

最小实例使用的是官方 `openai` Python SDK。

```bash
pip install openai
```

### 1.2 设置 API Key

先在终端里设置环境变量。

如果你要连 Ikun：

```bash
export IKUNCODE_API_KEY='你的 ikun key'
```

### 1.3 最小实例现在默认从环境变量读 key

打开：

- `minimal_openai_sdk_example.py`

你通常只需要改这项：

- `model`

因为示例现在默认会读取：

```python
os.getenv("IKUNCODE_API_KEY")
```

并且还会主动带上一个自定义 `User-Agent`，更接近这个仓库里已经跑通的请求方式。

### 1.4 为什么最小实例里额外加了 `default_headers`

你可能会发现，示例里除了 `api_key` 和 `base_url`，还多写了：

```python
default_headers={
    "User-Agent": "llmchat/0.1 (+local)",
}
```

这是有意加的，不是多余代码。

原因是：

- `openai` 官方 SDK 会自动帮你加认证头
  - `Authorization: Bearer ...`
  - `Content-Type: application/json`
- 但有些第三方兼容网关除了认证之外，还会根据请求头特征做额外拦截
- 这个仓库实测过，`ikuncode` 有时会拦掉 SDK 默认请求风格
- 显式补一个正常的 `User-Agent` 之后，请求就能通过

所以这里的 `default_headers` 不是为了替代认证，而是为了给所有请求统一补充默认请求头。

你可以把它理解成：

- `api_key`
  - 解决“你是谁”
- `default_headers`
  - 解决“你的请求长什么样”

如果你遇到类似报错：

```text
openai.PermissionDeniedError: Your request was blocked.
```

而你的 key 明明是对的，那么问题很可能不是认证失败，而是网关把请求头风格拦掉了。

这种情况下，补 `default_headers` 往往是有效的解决方式。

### 1.5 运行最小实例

```bash
python3 minimal_openai_sdk_example.py
```

如果配置正确，你会立刻看到一段模型输出。

这一步的目的只有一个：

- 先确认你的 API 是通的
- 先看到真实输出
- 这样后面再看复杂版本时，就不会觉得抽象

## 2. 最小实例的代码构成解释

最小实例文件是：

- `minimal_openai_sdk_example.py`

它的结构其实只有 4 步。

### 2.1 导入 SDK

```python
from openai import OpenAI
```

作用：

- 使用官方兼容客户端发请求

### 2.2 创建客户端

```python
client = OpenAI(
    api_key="sk-xxxx",
    base_url="https://api.ikuncode.cc/v1",
)
```

作用：

- 告诉程序：API key 是什么
- 告诉程序：请求要发到哪个地址

### 2.3 发起一次对话请求

```python
response = client.chat.completions.create(
    model="claude-sonnet-4-20250514",
    messages=[
        {"role": "system", "content": "你是一个有帮助的AI助手。"},
        {"role": "user", "content": "你好，请介绍一下自己。"}
    ],
    temperature=0.7,
    max_tokens=1024
)
```

这一步的重点字段是：

- `model`
  - 用哪个模型
- `messages`
  - 你发给模型的内容
- `temperature`
  - 回答更稳还是更发散
- `max_tokens`
  - 最多输出多长

### 2.4 打印模型输出

```python
print(response.choices[0].message.content)
```

作用：

- 把模型最终回答显示到终端

### 2.5 最小实例适合什么场景

适合：

- 只想确认接口通不通
- 只想快速发一条请求
- 想直接照着官方示例改

不太适合：

- 经常切换不同模型
- 想把参数放到配置文件里
- 想做交互式多轮聊天
- 想看真实 payload

## 3. 然后如何上手复杂的示例

复杂示例就是：

- `chat_with_config.py`

它更适合“长期反复使用”，因为模型参数都放在 `config` 文件里。

### 3.1 先准备配置文件

最简单的方法：

```bash
cp config/config-ikun.example.json config/config-ikun.json
```

然后按你的实际情况改：

- `base_url`
- `api_key_env`
- `model`

### 3.2 设置环境变量

```bash
export IKUNCODE_API_KEY='你的 ikun key'
```

### 3.3 先跑一个最短命令

单次提问：

```bash
python3 chat_with_config.py \
  --config config/config-ikun.json \
  --message "你好，请介绍一下自己。"
```

如果你想直接进入多轮聊天：

```bash
python3 chat_with_config.py --config config/config-ikun.json
```

### 3.4 复杂示例也要先“得到输出”

推荐你第一次只跑这条：

```bash
python3 chat_with_config.py \
  --config config/config-ikun.json \
  --message "你好"
```

先确认：

- config 能读到
- API key 生效
- 模型真的能返回内容

然后再去尝试：

- `--verbose`
- `--system`
- `--reasoning-effort`
- 交互式模式

### 3.5 常用命令

单次提问：

```bash
python3 chat_with_config.py \
  --config config/config-ikun.json \
  --message "请解释一下 Tensor Core 的作用"
```

交互式对话：

```bash
python3 chat_with_config.py --config config/config-ikun.json
```

临时覆盖系统提示词：

```bash
python3 chat_with_config.py \
  --config config/config-ikun.json \
  --system "你是一个严谨的中文 AI 助手" \
  --message "请解释 Triton 的优势"
```

查看真实请求：

```bash
python3 chat_with_config.py \
  --config config/config-ikun.json \
  --message "你好" \
  --verbose
```

## 4. 复杂实例和最小实例比，多在哪里，怎么实现的

这是最关键的一部分。

### 4.1 最小实例只有“一次硬编码请求”

`minimal_openai_sdk_example.py` 的特点是：

- key 写死在代码里
- base_url 写死在代码里
- model 写死在代码里
- prompt 写死在代码里
- 只发一次请求

优点：

- 非常短
- 非常直观

缺点：

- 每次切模型都要改代码
- 每次换 key 都要改代码
- 不适合长期反复调试

### 4.2 复杂实例把“硬编码”变成了“配置 + 参数”

`chat_with_config.py` 比它多出来的主要是这几类能力。

#### 4.2.1 配置文件读取

复杂实例会从：

- `config/config-ikun.json`

读取这些内容：

- `provider`
- `base_url`
- `api_key_env`
- `model`
- `temperature`
- `timeout_seconds`
- `model_reasoning_effort`

这样你以后切模型时，不用改 Python 代码，只改 config 即可。

#### 4.2.2 环境变量读取

复杂实例不是把 key 写死在代码里，而是：

- 从 `api_key_env` 指向的环境变量里取值

这样更安全，也更方便切换机器。

#### 4.2.3 交互式多轮聊天

最小实例只能发一轮。

复杂实例支持：

- 一直聊天
- 保留历史上下文
- `/clear`
- `/exit`

#### 4.2.4 兼容不同接口风格

复杂实例支持：

- `openai_compatible`
- `openai_responses`

所以它能适配的后端更多。

#### 4.2.5 兼容不同“思考强度”字段

不同后端字段不一样。

常见情况：

- GPT 类常用 `reasoning_effort`
- 某些 Claude 路径可能用 `output_effort`
- 某些 Gemini 路径可能用 `thinking_level`

复杂实例会帮你做一些映射和兼容，这也是它比最小实例长很多的重要原因。

#### 4.2.6 调试能力

复杂实例支持：

- `--verbose`

你可以直接看到：

- 实际调用的是哪个 endpoint
- 实际发出的 payload 长什么样

这对排查 API 问题非常有用。

### 4.3 一句话理解两者区别

你可以这样理解：

- `minimal_openai_sdk_example.py`
  - 面向“先跑通一次”
- `chat_with_config.py`
  - 面向“以后长期反复用”

## 5. config 最少需要哪些字段

最小可用配置是：

```json
{
  "llm": {
    "enabled": true,
    "label": "your-model",
    "provider": "openai_compatible",
    "base_url": "https://api.example.com/v1",
    "api_key_env": "YOUR_API_KEY",
    "model": "your-model-name"
  }
}
```

这几个字段含义如下：

- `enabled`
  - 是否启用
- `label`
  - 显示名称
- `provider`
  - 接口类型
- `base_url`
  - API 地址
- `api_key_env`
  - 环境变量名
- `model`
  - 模型名

## 6. 有哪些常用可选项

你通常会用到这些：

- `temperature`
- `timeout_seconds`
- `max_output_tokens`
- `model_reasoning_effort`
- `model_output_effort`
- `model_thinking_level`

如果你不确定后端支持哪种字段：

1. 先只写最小字段
2. 跑通
3. 再加 `--verbose`
4. 再补思考强度字段

## 7. 如何接入其他模型

最简单的方式：

```bash
cp config/config.example.json config/config-my-model.json
```

然后改这几个字段：

- `label`
- `provider`
- `base_url`
- `api_key_env`
- `model`

如果这个模型还有额外思考强度字段，再补上对应那一项。

## 8. 常见问题

### 8.1 为什么 `chat_with_config.py` 默认不限制输出长度

因为它是一个调试和直连工具。

默认不发 `max_output_tokens`，可以更接近后端原始行为。  
如果你要限制，再手动加：

```bash
--max-output-tokens 1200
```

### 8.2 为什么 `api_key_env` 不是直接写 key

因为更安全。

正确写法：

```json
"api_key_env": "IKUNCODE_API_KEY"
```

然后在终端里：

```bash
export IKUNCODE_API_KEY='你的 key'
```

### 8.3 多行 message 怎么输入

方法 1：直接换行

```bash
python3 chat_with_config.py \
  --config config/config-ikun.json \
  --message "请帮我做三件事：
1. 总结这段话
2. 提取重点
3. 给出建议"
```

方法 2：用 `$'...'`

```bash
python3 chat_with_config.py \
  --config config/config-ikun.json \
  --message $'请帮我做三件事：\n1. 总结这段话\n2. 提取重点\n3. 给出建议'
```

方法 3：从文件读入

```bash
python3 chat_with_config.py \
  --config config/config-ikun.json \
  --message "$(cat prompt.txt)"
```
